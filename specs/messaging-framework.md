# Messaging Framework

## Ziel
Performance-optimierte, vertrauenswürdige Plain-Emails für den deutschen Energiemarkt.

## Kernregeln
- keine Buttons/Banner/Newsletter-Optik
- einfache HTML-Struktur erhalten
- genau ein CTA-Link pro Mail
- keine fake-transaktionalen Betreffzeilen
- keine irreführende Dringlichkeit

## Betreffzeilen
- persönlich, konkret, nicht werblich
- selektive Nutzung von {{ contact.first_name }} und {{ opportunity.energieart }}
- Variation statt Einheitsmuster

Nicht verwenden (wenn nicht real):
- „Wir warten auf Rückmeldung“
- „Deine Anfrage“
- „Bitte dringend antworten“
- „Letzte Erinnerung“

## Preheader
- ergänzt den Betreff
- erhöht Relevanz
- kann eine sanfte offene Schleife setzen
- keine fake Support-/Reminder-Claims

## Zusätzliche Bridge-Zeile
Wenn Problem → CTA zu abrupt:
- genau 1 zusätzliche Zeile einfügen
- Zweck: Einwandbehandlung ODER Vertrauen ODER Relevanzbrücke
- nur selektiv, nicht in jeder Mail

## Szenario-Fokus
- spart: klare Kostenkonsequenz + Unsicherheitsabbau
- neueinzug: Grundversorgung + Timing deutlich
- spart_nicht: Ehrlichkeit + Monitoring + Fairness
- loop: Variation, Reaktivierung, mission-led referral

## Verbotene doppeldeutige Begriffe
- „dein aktueller Stand"
- „dein Tarif-Status"
- „deine Situation"
- „deine Daten"

## Signatur
Alle DE-Mails nutzen unter dem Gruß eine einheitliche kleine HTML-Signatur mit Brand, Logo, Adresse, Impressum und Datenschutz.
